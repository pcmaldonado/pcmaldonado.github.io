Customer Churn

Project Overview

Analysis

  • Customers are more likely to churn when tenure is low, when there have been complains and when they are single
  • People from certain city tiers are also more likely to churn than others
  • Buyers who mostly purchase from the Laptop & Accessory category are less likely to churn

By adressing these factors, (i.e. why does certain categories are associated with higher risk of churn), it is probably that churn would decrease.

Walkthrough

  • Analysed the data (distributions, type of data, missing values) and studied the relationship between the features and the average churn rate of the e-commerce
  • Preprocessed the data to be able to apply a machine learning model and used SMOTE to handle imbalanced data
  • Studied more than 10 classification algorithms (from Logistic Regression to ensemble methods like Random Forest, XGBoosting, and even Voting Classifiers)
  • Optimised the hyperparameters from the best performing models using 10-fold cross validation
    • Overall, the model with the best performance was the Voting Classifier based on the 4 best models fine-tuned and which weights were optimized using cross validation
  • Created a pipeline able to preprocess and predict on new data and used Pickle to import the best model
  • Used LIME to interpret results

Project information

  • Category: Classification
  • Data: It contains information on genre, tenure, preferred payment mode, and more, for over 5,600 customers of an e-commerce company (Kaggle)
  • Motivation: it is more expensive to acquire new customers than to keep old ones, thus preventing churn is vital for a company
  • Objective: Create a ML model able to detect customers likely to churn and understand why they are likely to churn, in order to decrease churn rate
  • Main Tools: Pandas, Sklearn, Matplotlib, Pickle, LIME, SMOTE
  • Code available on: Github
